1、栅格样式: 类container使用了媒体查询，左右有外边距；类container-fluid100%展示，为流式布局容器

  • 超小屏幕 手机(phones - xs) (<768px), 类前缀.col-xs-
  • 小屏幕 平板(tablets - sm) (≥768px), 类前缀.col-sm-
  • 中等屏幕 桌面显示器(desktops - md) (≥992px), 类前缀.col-md-
  • 大屏幕 大桌面显示器(larger desktops - lg) (≥1200px), 类前缀.col-lg-
  • 栅格样式使用原理分析：
    1. .col-xs-类样式全局，.col-sm-类样式写在最小宽度768px媒体查询里面， .col-md-类样式写在媒体最小宽度992px里面，.col-lg-类样式写在最小宽度1200px媒体查询中
    2. 因此写样式的时候，同一个标签如果需要同时存在几个类标签，书写格式依次为：col-xs-* col-sm-* col-md-* col-lg-*
    3. 这样写的好处就是当页面宽度大于1200px，起作用的是col-lg-*标签；页面大于992px，起作用的是col-md-*, 以此类推

The Bootstrap 3 grid system has four tiers of classes: xs (phones，extra small), sm (tablets), md (desktops), and lg (larger desktops). You can use nearly any combination of these classes to create more dynamic and flexible layouts.

Each tier of classes scales up, meaning if you plan on setting the same widths for xs and sm, you only need to specify xs.
